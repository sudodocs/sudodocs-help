# User Guide

This guide provides comprehensive instructions for technical writers, developers, and product managers using SudoDocs to automate documentation workflows.

* [Compose Single Note](single-note.md): Create a polished entry for a specific ticket.
* [Author a Feature](feature-author.md): Draft comprehensive feature documentation.
* [Check API Readiness](api-readiness.md): Validate OpenAPI specs and upload to ReadMe.
* [Generate Reference Docs](reference-docs.md): Convert Helm Charts and Specs to Markdown.
* [Audit Documentation](doc-drift.md): Detect technical drift between code and docs.
* [Product Assistant](chat.md): Use natural language to query your knowledge base.
