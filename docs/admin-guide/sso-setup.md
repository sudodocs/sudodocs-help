# Configure Single Sign-On (SSO)

Enterprise organizations can enforce authentication through identity providers like Okta, Azure AD, or Auth0 using OpenID Connect (OIDC). Single Sign-On is strictly an Enterprise feature.

## Prerequisites

* An active SudoDocs Enterprise subscription.
* System Administrator privileges.

## Configure your Identity Provider (IdP)

Create an OIDC application in your IdP and set the **Sign-in Redirect URI** to:
`https://app.sudodocs.com/login/sso/callback`

## Connect SSO to SudoDocs

1. Navigate to the **Admin Dashboard** and select **Security & SSO**.
2. Enter your **Corporate Domain** (e.g., `company.com`).
3. Enter the **Discovery URL (Issuer)**.
4. Provide the **Client ID** and **Client Secret** generated by your IdP.
5. Click **Save Configuration**.

> Once active, SudoDocs utilizes Just-In-Time (JIT) provisioning. Any user logging in with your corporate domain will automatically be routed through your IdP and provisioned as a Writer.
